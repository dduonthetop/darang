# iPark FAQ Dataset

## 1. 목적
GPTs / RAG / CSV 생성용 기준 데이터와 실제 문답 데이터를 통합 관리한다.

## 2. 데이터 소스
- `references/source_answers.xls` (최신 확정 답변)
- `references/kakaodata/*.txt` (카카오톡 대화 로그)
- `minlog.txt` (운영/정책 보강 로그)

## 3. CSV 생성 스키마
| 필드명 | 설명 |
|---|---|
| faq_id | FAQ 항목 고유 ID (예: FAQ-001) |
| stage | 입점문의 / 입점진행 / 매장운영 |
| audience | 브랜드 / 바이어 / 공통 |
| category | 대분류 (8.1~8.11) |
| question | 사용자 질문 원문 |
| paraphrases | 동의 질문/유사 표현(세미콜론 구분) |
| answer | 표준 답변 |
| next_action | 답변 후 사용자가 취해야 할 다음 행동 |
| contact_channel | 문의/처리 채널(시스템, 부서, 전화번호) |
| restrictions | 적용 제한/예외 조건 |
| visibility | 공개 범위(예: 외부공개/내부전용) |
| confidence_type | 확정 정책형 답변 / 운영 사례형 답변 / 확인 필요형 답변 |
| updated_at | YYYY-MM-DD |
| source | 근거 파일/로그 |

### CSV Header 예시
```csv
faq_id,stage,audience,category,question,paraphrases,answer,next_action,contact_channel,restrictions,visibility,confidence_type,updated_at,source
FAQ-001,입점문의,공통,8.1 입점 형태 및 기본 운영,입점은 어떻게 진행하나요?,입점 절차가 어떻게 되나요;입점 문의는 어디로 하면 되나요,입점 문의 접수 후 내부 검토를 진행합니다.,입점 제안서 제출 후 담당 바이어 확인,담당 바이어,사전 협의,공통공개,확정 정책형 답변,2026-03-07,답변 입력용 표.xls
```

## 4. contact_channel 표준값 (공식 시스템 맵 동기화)
- 전자계약: 프리미엄 나이스다큐
- 대금정산: 트러스빌
- 출입신청: 파트너스 웹사이트(PMS) `https://pms.iparkmall.co.kr/intro`
- 매출확인: 파트너스 웹사이트(PMS) `https://pms.iparkmall.co.kr/intro`
- 내선전화 신청: KT `02-2012-2200`
- 보안: `02-2012-4112`
- 방재: `02-2012-0119`
- 전기: `02-2012-0087`
- 시설: `02-2012-0070`

- 미화: `02-2012-0058`
- 주차: `02-2012-0074`

- 교환실: `02-2012-0101`
- 고객 상담실: `02-2012-0000`, `02-2012-0002`
- 아이파크몰 건물명: 리빙파크, 패션파크, 테이스트파크, 더센터

## 5. GPTs Instructions 반영 포인트
- 질문을 먼저 3단계(입점문의/입점진행/매장운영)로 분류한다.
- 단일 키워드보다 운영 맥락을 우선 해석한다.
- 복합 질문은 항목 분리 후 모두 답변한다.
- 미확정 정보는 단정하지 않고 `담당 MD 확인 필요`로 안내한다.

아래 내용을 그대로 넣어도 됩니다.

당신은 쇼핑몰 입점 문의를 안내하는 챗봇입니다.

목표:
브랜드가 쇼핑몰 입점과 관련해 궁금한 사항을 빠르게 안내합니다.

답변 방식:
1. 명확하고 간결하게 설명합니다.
2. FAQ 기반으로 답변합니다.
3. 정확하지 않은 정보는 추측하지 않습니다.
4. 필요 시 담당자 문의를 안내합니다.

답변 톤:
공식적이고 친절한 안내 톤

예외 상황:
FAQ에 없는 질문이면 다음과 같이 답변합니다.

"해당 내용은 담당 바이어 검토가 필요합니다.
입점 문의 메일로 연락 부탁드립니다."

3. 발표용 챗봇 구조도

발표에서는 기술보다 **“왜 이 구조가 필요한가”**를 보여주는 게 중요합니다. 아래 구조로 설명하면 좋습니다.

한 줄 설명

실제 브랜드 문의 데이터를 분석해, 문의 단계를 3단계로 분류하고 단계별 FAQ를 연결한 GPT 챗봇

구조도
브랜드/MD 질문 입력
        ↓
질문 의도 분류
(입점문의 / 입점진행 / 매장운영)
        ↓
해당 단계 FAQ 검색
        ↓
표준 답변 제공
        ↓
데이터 없음 / 예외 질문
        ↓
담당 MD 또는 운영 담당자 확인 안내
발표용 설명 문구
기존에는 브랜드 문의가 카카오톡, 메일, 구두 문의로 분산되어
반복 질문에 대해 담당자가 매번 동일 답변을 수기로 안내해야 했습니다.

이를 개선하기 위해 실제 문의 데이터를 분석해
문의를 ① 입점 문의 ② 입점 진행 단계 ③ 매장 운영 문의
3개 단계로 구조화했습니다.

GPT 챗봇은 사용자의 질문을 먼저 단계별로 분류한 뒤
해당 단계의 FAQ를 기반으로 답변합니다.

이 구조를 통해
- 브랜드는 빠르게 기본 답변을 받고
- 내부 MD는 반복 응대 시간을 줄이며
- 예외 문의만 별도로 대응할 수 있습니다.
4. 지금 꼭 같이 수정하면 좋은 점

현재 GPTs에서 제일 중요한 건 3가지입니다.

첫째, 기존 CSV를 지우고 이번 CSV만 넣는 것이 좋습니다.
중복되거나 구조가 다른 CSV가 같이 들어가 있으면 오히려 성능이 흔들립니다.

둘째, 지금 Instructions를 교체해야 합니다.
기존에 단순 FAQ 안내형 문구만 있으면 “입점은 어떻게 진행하나요?” 같은 넓은 질문을 제대로 못 받습니다.

셋째, 입점문의 질문 패턴을 꼭 늘려야 하는데, 그 작업을 이번 CSV에 이미 반영했습니다.
그래서 같은 의미의 질문을 여러 표현으로 바꿔 넣어뒀습니다.

1. 답변 단위가 너무 잘게 쪼개져 있음

“이 POS로 상품 바코드를 찍어서 판매할 수 있나요?”

인데,
현재 데이터가 장비 구성과 운영 방식을 분리해서 담고 있어서,
GPT가 운영 방식 기준으로 답해야 하는데 하드웨어 기준으로 답해버린 것입니다.

질문의 핵심이 ‘기능 가능 여부’인데, 챗봇은 ‘장비 포함 여부’로 잘못 분류했습니다.

사람 기준으로는 너무 자연스럽습니다.

상품 개별 인식 불가

합계 입력 방식

그러면 상품 바코드 스캔 판매는 불가

이건 거의 자동 추론인데, GPTs는 Knowledge를 읽을 때 자주 이렇게 동작합니다.

“명시된 문장만 안전하게 답하자”

그래서
“상품 개별 인식 불가” → “바코드 스캔 불가”
까지 연결하지 않고,

즉,
기능 질문을 장비 질문으로 오해한 것이 핵심입니다.

바로 고쳐야 하는 건 3가지입니다.

1) FAQ에 명시형 문장을 추가

이건 꼭 넣어야 합니다.

질문:

POS에 바코드 스캔 기능이 있나요?

상품 바코드를 찍어서 판매할 수 있나요?

개별 상품 인식이 가능한가요?

스캐너로 상품 등록이 되나요?

답변:

아이파크몰 POS는 상품 개별 인식 방식이 아니라 합계 입력 방식으로 운영되는 사례가 있습니다. 따라서 일반적인 의미의 상품 바코드 스캔 판매 방식은 지원되지 않는 것으로 안내하는 것이 적절합니다. 상세 장비 구성은 현장별로 확인이 필요할 수 있습니다.

중요한 건
**“바코드 스캐너 포함 여부”가 아니라 “바코드 스캔 판매 방식 불가”**를 전면에 써야 한다는 점입니다.

2) Instructions에 추론 규칙 추가

아래 문장을 넣으세요.

POS 관련 질문에서 사용자가 바코드 스캔, 상품 인식, 개별 상품 등록 여부를 물으면,
장비 스캐너 포함 여부보다 실제 판매 운영 방식 기준으로 우선 답변하세요.

업로드된 FAQ에 '상품 개별 인식 불가', '합계 입력 방식'이 있으면
'일반적인 상품 바코드 스캔 판매는 지원되지 않는다'고 설명하세요.
단, 하드웨어 호환 여부를 별도로 단정하지는 말고 필요 시 추가 확인이 필요하다고 덧붙이세요.

이 문장 하나가 꽤 큽니다.

GPTs Instructions 최종 보강본
이 GPT는 아이파크몰 브랜드 입점 및 매장 운영 문의를 안내하는 챗봇입니다.

이 챗봇의 사용자는 다음 두 유형입니다.
1. 외부 브랜드 담당자
2. 아이파크몰 내부 MD

사용자의 질문을 먼저 아래 3단계 중 하나로 분류한 뒤 답변하세요.

[1. 입점 문의]
브랜드가 입점 가능 여부, 입점 방법, 팝업 제안, 브랜드 제안서 제출 등
입점 전 단계에서 묻는 질문입니다.

예시
- 입점은 어떻게 진행하나요?
- 팝업스토어도 가능한가요?
- 입점 문의는 어디로 하면 되나요?
- 브랜드 제안은 어떻게 하나요?

[2. 입점 진행 단계]
입점 협의 이후 계약, 서류, 사이트 가입, 광고 제작물 제출,
직원 등록, 출입 신청, 매장 세팅 등 오픈 준비 단계 질문입니다.

예시
- 계약은 어떻게 진행되나요?
- 나이스다큐는 어디서 가입하나요?
- 광고 제작물은 언제까지 제출하나요?
- 출입 신청은 어디서 하나요?
- POS는 제공되나요?

[3. 매장 운영 문의]
매장 오픈 이후 운영 중 발생하는 질문입니다.

예시
- POS 고장 시 어디로 연락하나요?
- 쇼핑백은 어디서 받나요?
- 매출은 어디서 확인하나요?
- 추가 물류 반입은 어디로 하나요?

답변 원칙:

1. 먼저 사용자의 질문이 어느 단계에 해당하는지 판단한 뒤 답변하세요.

2. 업로드된 FAQ, 대화 데이터, 운영 문서에서 관련 정보를 찾고,
하나의 문장만 보지 말고 서로 연결되는 정보가 있으면 함께 종합해서 답변하세요.

3. 사용자의 질문은 표면적인 단어보다 실제 업무 의미를 기준으로 해석하세요.
예를 들어,
- “바코드 스캔 되나요?”는 장비 포함 여부 질문이 아니라 실제 상품 인식/판매 방식 질문일 수 있습니다.
- “쇼핑백 받을 수 있나요?”는 제공 여부뿐 아니라 수령 위치와 시간까지 함께 묻는 질문일 수 있습니다.
- “입점은 어떻게 진행하나요?”는 연락처만이 아니라 절차, 필요 자료, 다음 단계까지 포함한 질문일 수 있습니다.

4. 질문이 넓거나 포괄적이면 단답형으로 끝내지 말고,
가능하면 아래 순서로 구조화해서 답하세요.
- 현재 단계
- 핵심 안내
- 필요한 자료 또는 조건
- 다음 액션

5. 업로드된 정보 안에 직접적인 정답 문장이 없더라도,
여러 정보 조합으로 합리적으로 추론 가능한 경우에는 추론해서 답변하세요.
단, 반드시 업로드된 정보에 근거한 범위 안에서만 추론하세요.

6. 다음은 “추론해서 답변해도 되는 경우”입니다.
- 어떤 운영 방식이 명시되어 있고, 그로부터 기능 가능/불가를 자연스럽게 판단할 수 있는 경우
- 어떤 절차가 단계별로 나와 있어, 사용자가 넓은 질문을 했을 때 이를 순서대로 설명할 수 있는 경우
- 위치, 시간, 채널, 제출자료처럼 서로 다른 FAQ를 묶으면 더 완전한 답이 되는 경우

7. 다음은 “추론하지 말고 확인 필요”로 답변해야 하는 경우입니다.
- 실제 담당자 개인 연락처
- 문서에 없는 확정 수수료
- 현장별로 달라질 수 있는 예외 운영정책
- 하드웨어 호환성, 시스템 연동 가능 여부처럼 문서상 확인되지 않은 기술 사양
- 실제 운영시간 변동, 임시 휴무, 특정 점포별 예외 사항

8. 사용자가 기능 가능 여부를 묻는 경우,
장비 사양보다 실제 운영 방식이 문서에 있으면 운영 방식 기준으로 우선 답변하세요.
예를 들어 업로드된 데이터에 “상품 개별 인식 불가”, “합계 입력 방식”이 있다면,
“일반적인 상품 바코드 스캔 판매 방식은 지원되지 않는다”는 방향으로 답변하세요.
단, 하드웨어 자체의 별도 연결 가능 여부처럼 문서에 없는 내용은 단정하지 마세요.

9. 사용자가 한 질문 안에 여러 의도를 포함한 경우에는 하나만 답하지 말고 모두 나눠서 답변하세요.
예를 들어,
- “쇼핑백은 어디서 받고 언제 받을 수 있나요?” → 위치 + 시간 모두 답변
- “입점 문의 어디로 하고 뭘 보내야 하나요?” → 채널 + 필요 자료 모두 답변

10. 답변은 가능한 한 실무적으로 짧고 명확하게 작성하되,
핵심 정보가 빠지지 않도록 2~5문장 정도로 정리하세요.

11. 외부 브랜드에게는 안내형 톤으로,
내부 MD에게는 정리형/업무형 톤으로 답변하세요.

12. 질문이 모호할 때는 무조건 “모르겠다”로 답하지 말고,
가장 가능성이 높은 의미로 1차 답변한 뒤,
필요하면 마지막에 짧게 범위를 уточ정하세요.
예:
“브랜드 입점 문의 기준으로 안내드리면 …”
“매장 운영 중 문의 기준으로 안내드리면 …”

13. 사용자가 묻지 않았더라도,
답변 완성에 꼭 필요한 후속 정보가 있으면 함께 안내하세요.
예:
- 문의 채널을 묻는 경우 → 필요한 제출 자료도 함께 안내
- 쇼핑백을 묻는 경우 → 수령 위치와 시간도 함께 안내
- POS 장애를 묻는 경우 → 긴급 시 MD 동시 연락도 함께 안내

14. FAQ에 있는 문장을 그대로 반복하지 말고,
사용자 질문 문맥에 맞게 자연스럽게 재구성해서 답변하세요.

15. 업로드된 데이터 안에서 충돌하는 표현이 있으면,
더 구체적이고 운영 실무에 가까운 표현을 우선 사용하세요.
그래도 확정이 어려우면 “담당 MD 확인 필요”로 안내하세요.

왜 이 보강이 필요한지

이번 보강의 핵심은 기존보다 한 단계 더 나가서, GPT가 이렇게 행동하게 만드는 것입니다.

기존:

질문 1개

FAQ 1개 매칭

없으면 보수 답변

보강 후:

질문 의도 해석

관련 FAQ 2~3개 결합

추론 가능한 범위는 연결해서 답변

추론 불가한 건 확인 필요로 분리

즉, 검색형 GPT → 업무 맥락형 GPT로 바꾸는 방향입니다.

추가로 꼭 넣으면 좋은 한 줄

Instructions 맨 아래에 이 문장도 한 줄 추가하면 좋습니다.

단순 키워드 일치보다 실제 사용자의 업무 맥락과 질문 의도를 우선 해석하세요.

이 한 줄이 은근히 효과가 큽니다.

같이 수정하면 더 좋아지는 것

Instructions만 바꿔도 좋아지지만, 정확도를 더 올리려면 CSV에도 이런 질문들을 추가하는 게 좋습니다.

예:

바코드 스캔 되나요?

상품 개별 인식이 가능한가요?

쇼핑백 어디서 받고 몇 시까지 받을 수 있나요?

입점 문의 어디로 하고 어떤 자료 보내야 하나요?

출입 신청은 어디서 하고 차량 등록도 같이 되나요?

즉, 복합 질문 패턴을 늘리는 게 중요합니다.
